# Oracle - Database Editions

O Oracle Database é disponibilizado em diferentes editions (edições) para atender cenários que vão desde desenvolvimento individual até sistemas corporativos de missão crítica.
Cada edição possui limites e recursos, modelo de licenciamento e conjunto de funcionalidades diferentes.
# Enterprise Edition (EE): 

É a edição mais completa e poderosa da Oracle Database, destinada a ambientes corporativos de grande escala.
### Características principais:

- Conjunto completo de funcinalidades do banco de dados
- Suporte e recursos avançados de alta disponibilidade e escalabilidade
- Permite uso de diversos Database Options como Partitioning, Advanced Security e In-Memory
- Amplamente utilizada em bancos, telecomunicações e sistemas críticos.
- Licenciamento geralmente baseado em números de cores de CPU ou usuários

Essa edição é indicada para workloads que exigem alta performance, alta disponibilidade e grandes volumes de dados.
# 2. Standard Edition 2 (SE)

É uma edição mais simples e econônica voltada para aplicações de médio porte.

### Características Principais:

- Funcionalidades essenciais do Oracle Database
- Menos recursos avançados que a Enterprise Edition
- Limite de até 2 sockets de CPU por servidor
- Licenciamento mais simples e mais barato
- Adequada para departamentos, aplicações web e pequenas empresas

Essa edição atende muitos sistemas corporativos que não precisam dos recursos avançados do Enterprise Edition.

# 3.Express Edition (XE)

É a adição gratuíta do Oracle Database
### Caracteríticas principais:

- Uso gratuíto para desenvolvimento, teste e pequenas aplicações
- Limitações de CPU, memória e tamanho do banco de dados
- Ideal para aprendizado, protótipos e projetos pequenos
- Pode ser redistribuída com aplicações
Apesar das limitações, é uma opção popular para estudantes e desenvolvedores.

# 4.Personal Edition (PE)

É uma edição voltada para desenvolvimento individual em ambientes locais.

### Caracterísitcas principais:

- Uso em single user
- Compatibilidade com funcionalidades da Enterprise Edition
- Geralmente usada para desenvolvimento e testes locais
- Não inclui alguns recursos de cluster e gerenciamento

Serve com um ambiente completo para desenvolvedores que precisam testar aplicações Oracle localmente.

# O que é Oracle AI Database ?

O Oracle AI Database não é uma nova edition separada do banco de dados. Na prática, ele é o Oracle Database tradicional com recursos nativos de inteligência Artificial integrados.
Trata-se de uma evolução da plataforma existente, principalmente da Oracle Database Enterprise Edition, com funcionalidades adicionais voltadas para IA e aplicações modernas.

# Compreendendo o Que são Banco de Dados Relacionais

Em um banco de dados relacional, os dados são organizados em tabelas que se relacionam entre si através de chaves primarias e chaves estrangeiras, formando um esquema coeso e consistente.

As propriedades ACID (Atomicidade, Consistência, Isolamento e Durabilidade) garantem que as transações sejam executadas de forma confiável, mesmo em cenários de falha ou concorrência intensa. Essa garantia transacional é o que torna os bancos relacionais e escolha preferia para aplicações financeiras, sistemas de saúde, e-comerce e qualquer domínio onde a integridade dos dados é inegociável.
# Relational Database Management Systems (RDBMS)

Um sistema de Gerenciamento de Banco de dados Relacional (Relational Database Managment System - RDBMS) é o software responsável por criar, manter, consultar e administrar bancos de dados que seguem o modelo relacional.

O RDBMS atual como uma camada intermediária entre as aplicações e os dados físicos em disco, fornecendo mecanismos sofisticados de controle de concorrência, gerenciamento de transações, recuperação de falhas, segurança de acesso e otimização de consultas.

O Oracle Database, enquanto RDBMS, implementa um conjunto abrangente de funcionalidades que vão muito além do armazenamento e recuperação de dados.

Seu otimizador de consultas baseado em custo (Cost-Based Optimizer - CBO) analisa múltiplos planos de execução para cada instrução SQL e seleciona automaticamente o mais eficiente, considerando estatísticas de tabelas, índices disponíveis, distribuição de dados e recursos de hardware. Essa capacidade de auto-otimização é um dos grandes diferenciais do Oracle em relação a outros RDBMS do mercado.

Além das funções clássicas de um RDMS, o Oracle oferece recursos como Automatic Storage Management (ASM) para gerenciamento simplificado de discos, Automatic Workload Repository (AWR) para coleta automática de métricas de desempenho, e Automatic Database Diagnostic Monitor (ADDM) para diagnóstico proativo de problemas de performance.
#### O Que são e Qual o objetivo dos Esquemas (Schemas) ?

No Oracle Database, um schema é uma coleção lógica de objetos de banco de dados associados a um usuário específico.

Cada usuário do banco de dados possui um schema correspondente que contém os objetos criados por ele, como tabelas, índices, views, sequences, procedures, functions, triggers, packages e tipos de dados definidos pelo usuário. Essa organização em schemas permite separar logicamente os dados e a lógica de negócio de diferentes aplicações dentro do mesmo banco de dados.

As tabelas são os schemas objects, mais fundamentais, representando as estruturas que armazenam fisicamente os dados  em formato de linhas e colunas. Os índices são estruturas auxiliares que aceleram  recuperação de dados, funcionando de forma análoga ao índice remissivo de um livro.

As views fornecem uma camada de abstração sobre as tabelas, permitindo expor conjuntos de dados filtrados, agregados ou combinados sem duplicar informações. 
​Já as Materialized views armazenam fisicamente o resultado de consultas complexas, oferecendo ganhos significativos de performance para consultas analíticas.

Procedures, functions e packages são schema objects que encapsulam lógica de negócio diretamente no banco de dados, escritos em PL/SQL ou java. Esses objetos permitem centralizar regras de negócio, reduzir tráfego e rede entre aplicação e banco de dados e garantir consistência na execução de operações críticas. Triggers, por sua vez, são blocos de código que são executados automaticamente em resposta a eventos específicos, como inserções, atualizações ou exclusões de dados.

O domínio dos schemas objects e de como eles interagem entre si é essencial para projetar banco de dados eficientes e bem estruturados.
#### Structured Query Language (SQL)

É uma linguagem padrão universal para interação com banco de dados relacionais, permitindo que usuários e aplicações definam estruturas de dados (DDL), manipulem registros (DML), controlem permissões de acesso (DCL) e gerenciem transações (TCL).

Desde sua padronização pelo ANSI em 1986, o SQL evoluiu através de múltiplas revisões que incorporam funcionalidades como consultas recursivas, funções de janela (window functions), expressões de tabela comum (CTEs) e suporte a dados semiestruturados.
#### PL/SQL, Java e JavaScript

O PL/SQL (Procedure Language/SQL) é a linguagem procedural nativa do Oracle Database, projetada para combinar o poder declarativo do SQL com estruturas  de programação imperativa com variáveis, condicionais, laços de repetição e tratamento de exceções.

O PL/SQL executa diretamente dentro do engine do banco de dados, eliminando a latência de comunicação entre aplicação e servidor e permitindo o processamento de grandes volumes de dados com eficiência máxima. Procedures, functions, packages e triggers escritos em PL/SQL formam a espinha dorsal da lógica de negócio em milhares de sistemas corporativos ao redor do mundo.

Além do PL/SQL, o Oracle Database suporta Java como linguagem de programação server-side através da Oracle JVM (Java Virtual Machine) embarcada no próprio banco de dados. Stored procedures escritas em Java podem acessar diretamente os dados do banco, utilizar bibliotecas do ecossitema Java e implementar lógicas complexas que seriam difíceis de expressar puramente em PL/SQL.

Mais recentemente, o Oracle Database incorporou suporte a JavaScript através do Multilingual Engine (MLE), baseado no GraalVM. Com o MLE, desenvolvedores podem escrever stored procedure e funções em JavaScript que executam diretamente dentro do banco de dados, abrindo portas para uma enorme comunidade de desenvolvedores web e full-stack que agora podem contribuir com lógica server-side no Oracle sem precisar aprender PL/SQL. O suporte a módulos npm e à sintaxe moderna do ECMAScript torna essa integração ainda mais atrativa.

# Multitenant Architecture

A arquitetura multitenant, introduzida no Oracle Database 12c e refinada nas versões subsequentes, representa uma das inovações mais significativas na história do produto.

Nesse modelo, um único Container Database (CDB) pode hospedar múltiplos Pluggable Databases (PDBs), cada um funcionando com um banco de dados independente e isolado do ponto de vista das aplicações. O CDB gerencia os recursos compartilhados (memória, processos, redo logs) enquanto cada PDB mantém seus próprios schemas, objetos e configurações específicas.

Essa arquitetura traz benefícios substanciais para a consolidação de bancos de dados, pois permite que dezenas ou até centenas de PDBs compartilhem um única instância Oracle, reduzindo drasticamente o consumo de memória, CPU e armazenamento em comparação com o modelo tradicional de uma instância por banco de dados. A operação de plugging e unplugging permite mover PDBs entre diferentes CDBs com facilidade, tornando migrações, upgrades e provisionamento de ambientes tarefas significativamente mais simples.

No ponto de vista de segurança e isolamento, cada PDB opera como se fosse um banco de dados independente: usuários de um PDB não podem acessar dados de outro PDB, e cada PDB pode ter suas próprias políticas de autitoria, perfis de recurso e configurações de segurança. Ao mesmo tempo, o DBA pode gerenciar todos os PDBs de forma centralizada a partir do CDB root, executando operações de backup, patching e monitoramento em escala.

Na Oracle Cloud Infrastructure (OCI), a arquitetura multitenant é a base dos serviços de Autonomous Database, onde cada cliente recebe um PDB isolado dentro de uma infraestrutura compartilhada e gerenciada automaticamente pela Oracle. 

Compreender a arquitetura multitenant é essencial para qualquer DBA moderno, pois ela é o modelo padrão de implatação tanto on-premisses quanto em cloud.

# Processamento Analítico em Larga Escala

O Oracle Database in-Memory é uma das tecnologias mais poderosas para processamento analítico, permitindo armazenar dados em formato colunar diretamente na memória SGA. Enquanto o formato tradicional de linhas (row store) é otimizado para operações transacionais que acessam registros completos, o formato colunar (column store) é ideal para agregações, varreduras e cálculos analíticos que operam sobre um conjunto de colunas. O Oracle Database mantém ambos os formatos simultaneamente, atendendo cargas OLTP e OLAP na mesma instância.

O particionamento de tabelas é outra funcionalidade essencial para análises em larga escala, permitindo dividir tabelas muito grandes em partições menores baseadas em critérios  como faixa de datas, lista de valores ou hash. Com o partition prunning, o otimizador descarta automaticamente partições irrelevantes durante a execução de consultas, reduzindo drasticamente o volume de dados a ser processado. Combinado com o paralelismo (Parallel Query), o Oracle pode distribuir a carga de uma consulta analítica complexa entre múltiplos processos simultâneos, aproveitando ao máximo os recursos de hardware disponíveis.

Com a integração de capacidades de IA e Machine Learning diretamente no engine do banco de dados, o processamento analítico ganha uma nova dimensão: modelos preditivos podem ser treinados e executados sobre os dados sem movê-los para fora do banco, pipelines de feature engineering podem ser implementados em SQL e PL/SQL  e buscas por similaridade vetorial podem ser combinadas com filtros relacionais tradicionais em uma úncia consulta.

# Integração com Data Science e Analytics

O Oracle Database se posiciona como uma plataforma que integra capacidades de data science e analytics diretamente ao ambiente de banco de dados, eliminando a necessidade de mover dados para ferramentas externas.

O Oracle Machine Learning (OML) oferece uma série de algorítmos de Machine Learning (incluindo regressão, classificação, clustering, detecção de anomalias e associação) que podem ser invocados diretamente via SQL ou PL/SQL, operando sobre os dados in-place sem necessidade de extração.

# Oracle Cloud Infrastructure (OCI)

Uma característica importante da OCI é que sua arquitetura foi projetada com forte foco em performance, isolamento de recursos e segurança, algo especialmente relevante para cargas corporativas críticas, como banco de dados, sistemas financeiros e aplicações de grande escala.

É organizada em três grandes camadas de serviço:
#### Infraestrutura (IaaS)

A camada de infraestrutura fornece os recursos básicos de computação. Inclui:
- Máquinas virtuais e servidores bare metal
- Redes virtuais e balanceadores de carga
- Armazenamento em bloco, arquivo e objeto
- Clusters de alta performance

Essa camada permite que empresas criem ambientes completos de servidores na nuvem.
#### Plataforma (SaaS)

A OCI também oferece serviços gerenciados para desenvolvimento e operação de aplicações. Exemplos:

- Bancos de dados gerenciados como Oracle Autonomous Database
- Serviços de integração e mensageria
- Containers e Kubernetes
- Analytics e data integration

Esses serviços reduzem a necessidade de administração manual da infraestrutura.
#### Serviços de IA e Dados

Nos últimos anos a Oracle expandiu a OCI para incluir serviços avançados de dados e inteligência artificial, como:

- Modelos generativos
- Análise de dados em larga escala
- Machine Learning gerenciado
- Bancos de dados com vector search, como o Oracle AI Database

Outro apecto importante é a arquitetura global da OCI. A plataforma é composta por regiões distribuídas pelo mundo, Cada região possui múltiplos data centers isolados chamados availability domains, o que permite alta disponibilidade e tolerância a falhas.
