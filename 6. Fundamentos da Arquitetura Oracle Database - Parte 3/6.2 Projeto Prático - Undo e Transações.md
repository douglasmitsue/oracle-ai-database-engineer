### Missão 6

No Oracle Database, Undo e transações são conceitos centrais para garantir consistência, isolamento e recuperação de dados. Eles estão diretamente ligados às propriedades ACID, especialmente  atomicidade e consistência.

Transação é uma unidade lógica de trabalho composta por uma ou mais operações SQL (INSERT, UPDATE, DELETE). Ela representa um "bloco indivisível": ou todas as operações são confirmadas ou nenhuma é aplicada de forma permanente. Enquanto a transação está em andamento, suas alterações não são definitivas. Elas só tornam permanentes quando ocorre um COMMIT. Caso haja erro ou decisão explícita, um ROLLBACK desfaz todas as mudanças feitas desde o início da transação. Isso evita estados inconsistentes no banco.

Internamente, cada transação possui um identificador e mantém controle sobre os blocos de dados que está modificando. Outras sessões podem não ver essas mudanças imediatamente, dependendo do nível de isolamento, o que garante consistência de leitura.

O **Undo** é o mecanismo que permite "voltar atrás" nas alterações feitas por uma transação. Sempre que um dados é modificado, o Oracle não sobrescreve simplesmente o valor anterior. Ele grava uma versão anterior desse dado em uma estrutura chamada Undo Segment (ou tablespace UNDO). Esse registro contém informações suficientes para reverter a operação.

#### Na prática, o Undo tem três funções principais:

- Primeiro, permitir o ROLLBACK, desfazendo alterações de uma transação ativa ou abordada.
- Segundo, garantir consistência de leitura, implementando o chamado read consistency. Se uma consulta precisa ler dados que estão sendo modificados por outra transação, o Oracle usa o Undo para reconstruir uma versão consistente dos dados no tempo em que a query começou.
- Terceiro, suportar recovery, ajudando na recuperação após falhas. 
  Um ponto importante é que o Oracle usa o modelo de MVCC (Multi-Version Concurrency Control). Isso significa que múltiplas versões de um mesmo dado podem coexistir temporariamente. O undo é justamente o que permite manter essas versões antigas acessíveis sem bloquear leitura.

Resumindo de forma técnica: A transação define o escopo lógico das mudanças, enquanto o Undo armazena o estado anterior necessário para garantir rollback, isolamento e consistência temporal das leituras. Sem Undo, não haveria como implementar transações confiáveis nem consultas consistentes em ambientes concorrentes.
