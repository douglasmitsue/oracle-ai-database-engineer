
É uma unidade lógica de trabalho composta por uma ou mais operações SQL que devem ser executadas como um bloco indivisível.

Na prática, uma transação agrupa comandos como INSERT, UPDATE e DELETE, garantindo que eles sejam tratados de forma consistente pelo banco.

Ela segue o conceito clássico de ACID, que define o comportamento esperado:
- **Atomicidade**: ou tudo acontece, ou nada acontece
- **Consistência**: o banco sempre permanece em um estado válido
- **Isolamento**: transações não interferem umas nas outras
- **Durabilidade**: após confirmação, os dados não se perdem

**O ciclo de vida é simples**:
- A transação começa automaticamente ao executar o primeiro comando DML
- Você finaliza com COMMIT para confirmar as mudanças
- Ou usa ROLLBACK para desfazer tudo

Exemplo:
UPDATE contas SET saldo = saldo - 100 WHERE id = 1 ;
UPDATE contas SET saldo = saldo - 100 WHERE id = 2 ;
COMMIT;

Se algo der errado antes do COMMIT, um ROLLBACK garante que nenhuma alteração parcial fique no banco.

Transação é o mecanismo que garante a integridade e confiabilidade das operações no Oracle, especialmente em cenários críticos como sistemas financeiros, e é um dos pilares de qualquer SGBD relacional.
